Danke, dass du beitragen möchtest! Kurze Richtlinien, um schnell produktiv zu werden.

1. Forke das Repository und erstelle einen Branch pro Änderung (z. B. `feature/modal-edit`).
2. Schreibe klare Commit-Nachrichten.
3. Wenn du Logik änderst, füge passende Tests hinzu (Unit-Tests).
4. Öffne eine Pull Request und beschreibe kurz die Änderung, den Nutzen und mögliche Breaking Changes.
5. Bei größeren Änderungen zuerst eine Issue öffnen und kurz abstimmen.

Developer-Setup (lokal):

- Node.js installieren (empfohlen, falls du künftig Tooling hinzufügst): https://nodejs.org/
- Optional: VSCode + Live Server

Danke!
